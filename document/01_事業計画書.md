## 1. エグゼクティブサマリー

本事業は、誰でもオリジナルの絵本を投稿・閲覧できるモバイルプラットフォームアプリをiOS/Androidで提供するものである。

日本市場において「絵本の投稿・閲覧プラットフォーム」の直接競合はアルファポリスの「絵本ひろば」のみであり、SNS型コミュニティ機能・モバイルファーストの投稿体験・AI連携・クリエイターエコノミーのいずれにおいても大きな参入余地がある。

収益モデルは「クリエイター完全無料・読者サブスク課金」のnote型を採用。クリエイターの参入障壁を最小化してコンテンツを蓄積し、読者からのサブスクリプション収入で収益化する。

Phase 0（MVP）では、運営が用意した絵本を読者に提供する「閲覧専用アプリ」としてリリースし、UI/UXと閲覧体験の品質を検証した上で、Phase 1以降でクリエイター投稿・サブスク課金を段階的に導入する。

## 3. ビジネスモデル

### 3-1. 収益モデル：クリエイター無料・読者サブスク型（note型）

クリエイター（絵本作家志望）には全機能を完全無料で提供し、コンテンツの蓄積を促進する。収益は読者（親子）からのサブスクリプション課金を主軸とする。

### 3-2. クリエイター側（完全無料）

| 機能 | 説明 |
|------|------|
| 投稿 | 無制限、エディタ全機能、テンプレート |
| 成長支援 | アクセス解析、フォロワー通知、コメント、ランキング |
| 収益化 | 投げ銭受取（手数料15%）、有料コンテンツ設定、印刷注文（実費のみ） |

### 3-3. 読者側（サブスク課金）

| プラン | 月額 | 主な機能 |
|--------|------|---------|
| Free | 0円 | 1日5冊まで閲覧、いいね・コメント、フォロー5人まで、広告表示 |
| Standard | 580円 | 無制限閲覧、AI読み聞かせ、オフライン保存30冊、広告非表示 |
| Family | 980円 | Standard全機能＋子どもプロフィール3人、読書記録、レコメンド、新作先行公開 |

### 3-4. 4つの収益源

1. **読者サブスク（メイン）** — Standard 580円 / Family 980円
2. **広告収入** — Free層への広告配信
3. **プラットフォーム手数料** — 投げ銭・有料コンテンツの15%
4. **印刷注文マージン** — 製本注文の中間マージン

## 4. プロダクト設計

### 4-1. Phase 0（MVP）の定義

**最初のリリースは「運営が用意した絵本を読む閲覧専用アプリ」とする。**

クリエイター投稿機能・サブスク課金はPhase 1以降で導入する。

#### 検証項目

- 「この絵本アプリを使いたいか」（DL数・DAU・継続率）
- 「どの絵本に反応が集まりそうか」（レビュー・インタビュー・運営観察）
- 「UI/UXとして快適か」（レビュー・離脱率・セッション時間）

#### Phase 0 に入れる機能

| 機能 | 説明 | 対象 |
|------|------|------|
| ホームフィード | おすすめ・にんきランキングの絵本一覧 | 読者 |
| 絵本ビューア | スワイプでページ送り（iPhone / iPad / Android対応） | 読者 |
| いいね | お気に入りの絵本をローカル保存 | 読者 |
| 閲覧履歴 | 読んだ絵本の履歴をローカル保存 | 読者 |
| マイページ | 閲覧履歴といいね一覧の確認 | 読者 |
| マルチデバイス対応 | iPhone + iPad + Android（Flutter） | 共通 |

#### Phase 0 で作らないもの

| 機能 | 導入時期 | 理由 |
|------|---------|------|
| サブスク課金 | Phase 1 | まず閲覧体験の品質を検証してから収益化 |
| 1日5冊制限 + ペイウォール | Phase 1 | サブスク導入と同時に実装 |
| 認証 | Phase 1 | Phase 0 は認証なしの最小構成で検証する |
| ブックマーク | Phase 1 | Phase 0 はいいねと閲覧履歴に絞る |
| 管理画面 | Phase 1 | Phase 0 は Firebase Console と R2 手動運用で代用 |
| クリエイター投稿エディタ | Phase 1 | Phase 0の反応を見てから開放 |
| フォロー機能 | Phase 1 | クリエイター投稿開放と同時に意味を持つ |
| プッシュ通知 | Phase 1 | フォロー機能と連動 |
| 検索・カテゴリ | Phase 2 | コンテンツ量が増えてから |
| AI読み聞かせ音声 | Phase 2 | API費用がDAUに比例するため収益安定後 |
| オフライン保存 | Phase 2 | サブスク限定機能として |
| 投げ銭・有料コンテンツ | Phase 3 | クリエイターエコノミーの成熟後 |
| 印刷注文 | Phase 3 | 外部連携が必要 |

### 4-2. 画面構成（Phase 0）

- タブバー構成：Home / My（Phase 0は2タブのみ）
- ホームフィード：おすすめ / にんきランキングの絵本一覧
- 絵本ビューア：スワイプページ送り、いいねボタン、作者情報表示
- マイページ：閲覧履歴一覧、いいね一覧
- ※Phase 1以降で Search / Create タブを追加

### 4-3. UI/UX方針

- 絵本の世界観を邪魔しない、温かみのあるミニマルデザイン
- スマホ（縦）とタブレット（横可）の両方で快適な閲覧体験
- ページ送りは直感的なスワイプ操作、ページインジケーター表示
- 子どもが触っても安全なUI（広告導線なし、課金導線なし＝Phase 0）

## 5. 技術設計

### 5-1. 技術スタック

| コンポーネント | 技術 | 選定理由 |
|--------------|------|---------|
| フロントエンド | Flutter (Dart) | iOS/Android同時開発、1人開発に最適、タブレット対応が容易 |
| データベース | Cloud Firestore | リアルタイム同期、セキュリティルール、無料枠が大きい |
| ローカル保存 | shared_preferences / Hive | Phase 0 の いいね / 閲覧履歴を認証なしで保持できる |
| 画像保存・配信 | Cloudflare R2 | エグレス（転送）料金が完全無料、CDN内蔵、S3互換API |
| Firebase Auth | Phase 1 以降 | 認証導入後にユーザーデータ同期へ拡張する |
| Cloud Functions | Phase 1 以降 | いいね / 履歴のサーバー同期や運営向け機能で利用する |
| プッシュ通知 | Firebase Cloud Messaging | Phase 1で導入、Flutter SDKと統合 |
| サブスク課金 | RevenueCat | Phase 1で導入、月間$2,500収益まで無料 |
| 小さい画像 | Firebase Storage | アバター等の軽量ファイルに限定使用 |

### 5-2. Cloudflare R2 を選定した理由

絵本アプリは画像がメインコンテンツであり、読者が閲覧するたびに大量の画像転送（エグレス）が発生する。Firebase Storageの転送料金$0.12/GBでは、DAU 5,000人で月額約6万円のコストが発生する。

Cloudflare R2はエグレスが完全無料のため、同じDAU 5,000人でも月額約500円で済む。この差は事業の成否を左右するレベルのインパクトがある。

#### R2のリスクと対策

- **障害実績** — 2025年に2回のグローバル障害（各約1時間）が発生。年数回のダウンタイムは想定すべき
- **対策1** — Cache-Controlヘッダーで長めのキャッシュを設定し、CDNヒット率を上げる
- **対策2** — アプリ側に画像読み込み失敗時のリトライ＋プレースホルダー表示を実装
- **対策3** — S3互換APIのため、万が一の値上げ時にBackblaze B2等への移行が容易

### 5-3. Firestoreデータモデル（Phase 0）

| コレクション | 主要フィールド | 用途 |
|-------------|--------------|------|
| books/{bookId} | title, description, authorName, coverUrl, ageGroup, pageCount, likeCount, viewCount, status, pages, createdAt | 絵本メタデータとページ情報 |

Phase 0 では Firestore は閲覧用の `books` コレクション取得に限定する。
いいねと閲覧履歴はローカル保存とし、`users` / `likes` / `history` / `bookmarks` は Phase 1 以降で導入する。

## 8. リスクと対策

| リスク | 影響度 | 対策 |
|-------|-------|------|
| コンテンツ不足（鶏と卵問題） | 高 | Phase 0で運営が絵本を用意。Phase 1ではクリエイターコミュニティへの積極的なアウトリーチを実施 |
| サブスクCVRが想定を下回る | 中 | Free層の広告収入と投げ銭手数料で補完。価格帯の調整余地あり |
| Cloudflare R2の障害 | 中 | CDNキャッシュ戦略＋アプリ側リトライ＋プレースホルダー表示で影響軽減 |
| App Store審査リジェクト | 中 | Phase 0は運営投稿のみのためUGCガイドライン要件が緩い。Phase 1でコンテンツ報告機能を必ず実装 |
| 絵本ひろばとの競合激化 | 低 | 絵本ひろばは完全無料で持続性に疑問。SNSコミュニティ機能とUI/UX品質で差別化 |
| R2の値上げ | 低 | S3互換APIのためBackblaze B2等への移行が容易。ベンダーロックインリスクは低い |

## 9. まとめ

本事業は、日本市場において明確な競合空白がある「絵本の投稿・閲覧プラットフォーム」に参入するものである。

Phase 0（MVP）として、運営が用意した絵本を閲覧できるアプリをiOS/Androidで同時リリースし、UI/UXの品質と閲覧体験の市場適合性を検証する。技術基盤はFlutter + Firebase + Cloudflare R2の構成により、初年度の運用コストを約2〜3万円に抑えつつ、DAU 10,000人規模まで月額2万円以下でスケール可能な設計としている。

Phase 1以降でクリエイター投稿の開放とサブスク課金を段階的に導入し、DAU 5,000人・CVR 10%の想定で月間営業利益約24万円の到達を目指す。

海外ではStorybirdが900万ユーザー・3,000万作品の規模に成長した実績があり、日本語圏に特化したモバイルファーストの同等サービスとして、十分な成長ポテンシャルがあると判断する。
