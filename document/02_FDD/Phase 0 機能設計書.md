# えほんのもり - Phase 0 機能設計書

## 概要

Phase 0（MVP）は「閲覧体験の検証」を目的とした最小構成のアプリ。
運営が用意した20〜30冊の絵本を、認証なしで自由に閲覧できる。

本資料を Phase 0 実装の source of truth とする。
Phase 0 では認証、ブックマーク、管理画面は実装しない。

---

## 対象プラットフォーム

- iOS（iPhone + iPad）
- Android（スマートフォン + タブレット）
- Flutter で同時開発

---

## 画面一覧

| 画面 | 概要 |
|---|---|
| ホーム画面 | 絵本一覧（おすすめ + ランキング） |
| 絵本ビューア | 全画面スワイプ式ビューア |
| マイページ | 閲覧履歴 + いいね一覧 |

### 作らない画面（Phase 1以降）

- 認証画面（Phase 0は認証なし）
- 管理画面（Firebase Consoleで代用）
- ブックマーク画面
- 検索画面
- クリエイター投稿画面
- 設定画面

## 1. ホーム画面

### 1.1 概要

アプリのメイン画面。絵本の一覧を「おすすめ」と「にんきランキング」の2セクションで表示する。

### 1.2 データ取得

- **タイミング**: アプリ起動時に1回のみ
- **取得元**: Firestore `books` コレクション（`status == "published"`）
- **取得方法**: 全件取得（Phase 0は20〜30冊のため）
- **フィルタリング・ソート**: クライアント側で実行（追加のFirestore読み取りなし）

### 1.3 画面構成

全体が1つのスクロール領域（ヘッダー・フィルターも一緒にスクロール）。

#### ヘッダー
- アプリ名「えほんのもり」（左）
- サブテキスト「すてきな えほんに であおう」
- プロフィールアイコン（右）→ タップでマイページへ

#### 年齢フィルター
- 横並びボタン: 「すべて」「0〜2さい」「3〜5さい」「6さい〜」
- デフォルト: 「すべて」が選択状態
- タップで即座にフィルタリング（おすすめ・ランキング両方に適用）
- クライアント側フィルタリングのため通信なし

#### おすすめセクション
- セクションタイトル: フィルター選択に応じて変化
  - 「すべて」→「おすすめ」
  - 「0〜2さい」→「0〜2さいの おすすめ」等
- 横スクロールカードリスト
- 並び順: `createdAt` 降順（新着順）固定
- 表示件数: 最大5冊
- カード表示: 表紙画像 + タイトル + 作者名

#### にんきランキングセクション
- セクションタイトル: ★アイコン +「にんきランキング」
- 縦リスト表示
- 並び順: `likeCount` 降順
- 表示件数: フィルター適用後の全件
- カード表示: 順位バッジ + 表紙サムネイル + タイトル + 作者名 + いいね数
- 順位バッジ: 1〜3位は色付き（アクセント/ミント/ピーチ）、4位以降はグレー

### 1.4 レスポンシブ対応

| 要素 | スマホ | タブレット |
|---|---|---|
| おすすめカード幅 | 120px | 150px |
| おすすめカード高さ | 140px | 170px |
| ランキングサムネイル | 48×56px | 56×64px |
| フォントサイズ（タイトル） | 11px | 13px |
| 左右パディング | 16px | 28px |

### 1.5 状態管理

| 状態 | 表示 |
|---|---|
| ローディング中 | スケルトンUI（カード形状のプレースホルダー） |
| データ取得成功 | 通常表示 |
| データ取得失敗 | 「えほんを よみこめませんでした」+ リトライボタン |
| 絵本0件（フィルター結果） | 「この ねんれいの えほんは まだ ありません」 |

## 2. 絵本ビューア

### 2.1 概要

全画面表示のスワイプ式ビューア。タップ→即座に全画面で絵本が開く（詳細画面は挟まない）。

### 2.2 データ取得

- **タイミング**: ビューア表示時
- **取得内容**: `pages` 配列内の各 `imageUrl` をR2から取得
- **読み込み戦略**: 現在のページ + 前後1ページを優先プリロード
- **キャッシュ**: 取得済み画像はメモリキャッシュに保持

### 2.3 ページ構成

```
[表紙] → [ストーリーページ 1] → ... → [ストーリーページ N] → [おしまい画面]
```

#### 表紙（page 0）
- 表紙画像（大）
- タイトル（多言語: 日本語優先）
- 作者名
- あらすじ（多言語: 日本語優先）
- メタ情報: ページ数 + 対象年齢

#### ストーリーページ（page 1〜N）
- ページ画像（全画面背景）
- テキスト表示:
  - `text` がnull → 画像のみ表示（テキスト焼き込み画像）
  - `text` がある → `textPosition` に従ってオーバーレイ表示
    - `"top"`: 画面上部に半透明背景付きテキスト
    - `"bottom"`: 画面下部に半透明背景付きテキスト
    - `"center"`: 画面中央にテキスト
    - `null`: デフォルトで `"bottom"` 扱い
- タブレット横向き: イラストとテキストを横並びレイアウト

#### おしまい画面（最終ページ後）
- 「おしまい」テキスト
- 表紙サムネイル（小）
- タイトル + 作者名
- いいねボタン（大きめ、目立つ）
- 「ホームにもどる」リンク

### 2.4 操作方法

#### ページ送り
- **左右スワイプ**: 右スワイプで次ページ、左スワイプで前ページ
- **タップ**: 画面右40%タップで次ページ、左35%タップで前ページ
- **表紙から前ページ**: 無効（何も起きない）
- **おしまい画面から次ページ**: 無効

#### オーバーレイUI
- **表示トリガー**: 画面中央タップ、または上端12%/下端12%タップ
- **非表示**: 3秒後に自動フェードアウト（300msアニメーション）
- **UI要素**:
  - 左上: 戻るボタン（半透明円形、タップでホームに戻る）
  - 中央上: ページ番号「3 / 12」（半透明ピル型）
  - 右上: いいねボタン（半透明円形、ハートアイコン）
- **ナビゲーション矢印**: UI表示中のみ左右に半透明矢印ボタン表示
- **ページドット**: UI非表示中のみ画面下部にドットインジケーター表示

#### おしまい画面
- オーバーレイUIは非表示
- いいねボタンと「ホームにもどる」は常時表示

### 2.5 いいね機能

- **保存先**: ローカル（SharedPreferences）
- **データ形式**: `Set<String>` で bookId の集合を保持
- **トグル動作**: タップでいいね追加/解除を切り替え
- **表示**: ハートアイコンが塗りつぶし（いいね済み）/ アウトライン（未いいね）に変化
- **likeCount反映**: Firestoreの `books.likeCount` は更新しない（Phase 0ではローカルのみ）
- **ランキングへの影響**: Phase 0のランキングはFirestoreの `likeCount` 基準のため、ローカルいいねはランキングに反映されない

### 2.6 閲覧履歴記録

- **タイミング**: ビューアを開いた時点で記録
- **保存先**: ローカル（SharedPreferences / Hive）
- **データ形式**: `List<Map>` で `{ bookId, readAt, lastPage }` を保持
- **重複処理**: 同じ bookId が既にあれば `readAt` と `lastPage` を上書き更新
- **保存上限**: なし（全件保持）
- **lastPage更新**: ページをめくるたびに更新

### 2.7 レスポンシブ対応

| 要素 | スマホ | タブレット縦 | タブレット横 |
|---|---|---|---|
| 表紙イラストサイズ | 140px | 200px | 180px |
| ストーリーテキストサイズ | 14px | 22px | 20px |
| テキスト行間 | 1.8 | 2.0 | 2.0 |
| レイアウト方向 | 縦並び | 縦並び | 横並び（イラスト+テキスト） |
| オーバーレイボタンサイズ | 32px | 40px | 40px |

### 2.8 状態管理

| 状態 | 表示 |
|---|---|
| 画像読み込み中 | ページ背景色 + ローディングスピナー |
| 画像読み込み成功 | 通常表示 |
| 画像読み込み失敗 | プレースホルダー画像 + リトライボタン |

## 3. マイページ

### 3.1 概要

閲覧履歴といいねした絵本を確認できる画面。データは全てローカル保存。
ブックマーク機能は Phase 0 の対象外とする。

### 3.2 画面構成

#### ヘッダー
- 「← ホーム」戻るボタン（左）
- 「マイページ」タイトル（中央）

#### タブ
- 「りれき (件数)」（デフォルト選択）
- 「いいね (件数)」

#### りれきタブ
- 表示形式: 縦リスト
- 各行: サムネイル + タイトル + 作者名 + 閲覧時刻
- 並び順: `readAt` 降順（新しい順）
- タップ: ビューアに遷移（`lastPage` は使わない、常に表紙から）
- 空状態: 「まだ よんだ えほんが ありません。ホームから えほんを えらんでみよう。」

#### いいねタブ
- 表示形式: カードグリッド
- 各カード: 表紙画像 + タイトル + 作者名
- 並び順: いいねした順（新しい順）
- タップ: ビューアに遷移
- 空状態: 「まだ いいねした えほんが ありません。すきな えほんに いいねしよう。」

### 3.3 レスポンシブ対応

| 要素 | スマホ | タブレット |
|---|---|---|
| いいねグリッド列数 | 2列 | 3列 |
| サムネイルサイズ（履歴） | 48×56px | 56×64px |
| カード高さ（いいね） | 120px | 150px |

### 3.4 データ管理

- **データ元**: 全てローカル（SharedPreferences / Hive）
- **表示用データの組み立て**: ローカルの bookId リストと、起動時に取得済みの books データを突合して表示
- **存在しない bookId**: 運営がFirestoreから絵本を削除した場合、該当の履歴/いいねは非表示にする（ローカルデータは残る）
- **リセット機能**: Phase 0ではなし（アプリ削除でリセット）

## 4. データ投入（運営作業）

### 4.1 概要

Phase 0では管理画面を作らず、Firebase ConsoleとR2ダッシュボードから直接データを投入する。

### 4.2 画像アップロード手順

1. 絵本の画像を用意（WebP形式推奨）
   - 表紙: 600×800px
   - ページ: 1200×1600px
2. Cloudflare R2ダッシュボードまたはCLI（wrangler）でアップロード
   ```
   ehonnomori/covers/{bookId}.webp
   ehonnomori/pages/{bookId}/001.webp
   ehonnomori/pages/{bookId}/002.webp
   ...
   ```
3. アップロード後のURLを控える

### 4.3 Firestoreデータ投入

Firebase Consoleから `books` コレクションにドキュメントを作成。

投入用JSONテンプレート:
```json
{
  "title": {
    "ja": "くもの子マルの旅",
    "en": null
  },
  "description": {
    "ja": "そらを たびする くもの子マルの ぼうけん",
    "en": null
  },
  "authorName": "えほんのもり",
  "coverUrl": "https://r2.example.com/ehonnomori/covers/book001.webp",
  "ageGroup": "0-2",
  "pageCount": 10,
  "likeCount": 0,
  "viewCount": 0,
  "status": "published",
  "pages": [
    {
      "imageUrl": "https://r2.example.com/ehonnomori/pages/book001/001.webp",
      "text": {
        "ja": "むかしむかし、空のうえに ちいさな くもの子が いました。",
        "en": null
      },
      "textPosition": "bottom"
    },
    {
      "imageUrl": "https://r2.example.com/ehonnomori/pages/book001/002.webp",
      "text": null,
      "textPosition": null
    }
  ],
  "createdAt": "2026-XX-XXTXX:XX:XXZ",
  "updatedAt": "2026-XX-XXTXX:XX:XXZ"
}
```

### 4.4 投入チェックリスト

各絵本について以下を確認:
- [ ] 表紙画像がR2にアップロード済み
- [ ] 全ページ画像がR2にアップロード済み
- [ ] `coverUrl` がアクセス可能
- [ ] 全ページの `imageUrl` がアクセス可能
- [ ] `pageCount` と `pages` 配列の長さが一致
- [ ] `ageGroup` が "0-2" / "3-5" / "6+" のいずれか
- [ ] `status` が "published"
- [ ] `title.ja` と `description.ja` が入力済み

## 5. ローカルデータ仕様

### 5.1 保存先

Flutter の `shared_preferences`（軽量データ）または `hive`（構造化データ）を使用。

### 5.2 データキーと形式

#### いいね
- キー: `liked_book_ids`
- 形式: `List<String>`（bookIdのリスト）
- 例: `["book001", "book003", "book007"]`

#### 閲覧履歴
- キー: `reading_history`
- 形式: `List<JSON文字列>`
- 各要素:
  ```json
  {
    "bookId": "book001",
    "readAt": "2026-04-01T10:30:00Z",
    "lastPage": 5
  }
  ```
- 並び順: `readAt` 降順（新しい順が先頭）
- 重複: 同一 `bookId` は1件のみ保持（新しい閲覧で上書き）

## 6. 非機能要件

### 6.1 パフォーマンス

- アプリ起動からホーム表示まで: 3秒以内（Wi-Fi環境）
- ビューア表示から表紙表示まで: 1秒以内
- ページ遷移アニメーション: 300ms以内

### 6.2 オフライン対応

- Phase 0ではオフライン対応なし
- 通信エラー時は各画面のエラー状態を表示

### 6.3 画像最適化

- フォーマット: WebP
- 表紙: 600×800px、100KB以下目安
- ページ: 1200×1600px、200KB以下目安
- CDNキャッシュ: R2のキャッシュヘッダーで制御

### 6.4 対応OS

- iOS 15.0以上
- Android 8.0（API 26）以上
